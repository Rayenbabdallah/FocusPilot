# CODE2CURE Submission Checklist (Phase 2)

Based on `Technical-Challenge.pdf` Phase 2 requirements.

## Required Deliverables

- [x] Public GitHub repository with code + clear README
- [x] Functional prototype (frontend + backend runnable)
- [x] Technical report (`docs/technical_report.pdf`)
- [ ] 7-minute presentation deck (external artifact)
- [ ] 3-minute demo video (external artifact)

## Repository Quality Checks

- [x] `README.md` includes setup and run instructions
- [x] `LICENSE` file present
- [x] `.env` ignored from Git
- [x] Backend tests pass (`python -m pytest -q`)
- [x] Frontend build passes (`npm run build`)

## Before Final Submission

1. Ensure the repo is public.
2. Ensure no commits are pushed after submission deadline.
3. Add links/attachments for:
   - final 7-minute presentation deck
   - final 3-minute demo video
4. Verify `docs/technical_report.pdf` is committed (or provide the PDF through the official submission form).
